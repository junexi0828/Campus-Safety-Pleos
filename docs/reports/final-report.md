## 1. 프로젝트 개요

### 1.1 프로젝트명
**교내 안전 통합 관리 플랫폼 (Smart Campus Safety Management Platform)**

### 1.2 프로젝트 목표
현대자동차 그룹의 Pleos Connect SDK 기반 Android Automotive OS(AAOS)를 활용하여 대학 캠퍼스의 안전을 종합적으로 관리하는 실시간 안전 응답 플랫폼 개발

### 1.3 핵심 콘셉트
**"사용자 위험신호 → 즉시 순찰차 알림 → 실시간 위치 확인 및 대응"**

- 학생 및 캠퍼스 이용자가 위험/긴급 상황을 앱으로 신고
- 순찰 차량 운전자가 실시간으로 신고 알림과 신고자 위치 수신
- 안전 관리자는 웹 대시보드에서 전체 상황 통합 모니터링

### 1.4 주요 기능 (순서별 우선순위)

**Phase 1: 핵심 기능 (필수 MVP)**
1. **위험 신호 보내기** - 학생이 앱으로 위험 상황 신고 (사진, 위치, 설명)
2. **실시간 순찰 차량 위치 추적** - GPS 기반 위치 실시간 업데이트
3. **순찰 차량 알림 수신** - 신고 즉시 순찰차 앱에 푸시 알림
4. **현황 대시보드** - 행정안전부 담당자의 통합 모니터링 화면
5. **대응 상태 관리** - 신고 → 접수 → 현장 출동 → 처리 완료 추적

**Phase 2: 확장 기능 (선택 사항)**
- 순찰 차량의 환경 데이터(외부 온도, 차량 내부 HVAC) 수집
- 민원 관리 시스템 (포트홀, 시설 파손 등)
- 예측 기반 순찰 경로 최적화

### 1.5 기대 효과
- **응답 시간**: 평균 5분 이내 순찰차 현장 도착 (기존 대비 70% 단축)
- **사용자 만족도**: 안전 체감도 향상
- **운영 효율성**: 데이터 기반 순찰 계획 수립
- **산업적 의미**: Pleos 플랫폼의 혁신적 활용 사례 창출

## 2. 프로젝트의 의의 및 필요성

### 2.1 기술적 배경
- Android Automotive OS(AAOS)는 차량 산업의 표준 플랫폼으로 부상 중
- 현대자동차 Pleos Connect SDK는 차량 제어, GPS 위치 추적, 실시간 데이터 동기화 API 제공
- Fused Location SDK와 Sync SDK를 통한 실시간 위치 추적 및 데이터 동기화 기술 활용 가능

### 2.2 현실적 필요성
- 대학 캠퍼스에서 발생하는 긴급 상황(범죄 의심, 사고, 보안 위협 등)에 신속한 대응 필요
- 순찰 차량과 신고자 간 정보 격차 해소 필요
- 위험 상황의 실시간 추적 및 관리 시스템 부재

### 2.3 학술적 및 산업적 가치
- **기술 혁신**: 자동차 OS 기술을 캠퍼스 안전 도메인에 응용한 선제적 사례
- **인력 양성**: 모빌리티 산업 진출 희망 학생의 실무 역량 강화
- **산업 협력**: 현대자동차와의 기술 협력 및 공모전 수상 기회
- **사회 기여**: 캠퍼스 안전 문화 확산 및 학생 보호

## 3. 핵심 기능 및 기술 구성

### 3.1 사용자별 주요 기능

| 사용자 유형 | 주요 기능 | 사용 기기 |
|-----------|---------|---------|
| **학생/캠퍼스 이용자** | 위험신호 송출, 위치 공유, 상태 조회 | 모바일 앱 (Android) |
| **순찰 차량 운전자** | 신고 알림 수신, 신고자 위치 확인, 대응 상태 업데이트 | 차량 앱 (Pleos Emulator/실제 차량) |
| **행정안전부 담당자** | 통합 모니터링, 신고 현황 관리, 순찰차 배치 최적화 | 웹 대시보드 |
| **기숙사/당직자** | 당번 별 알림 수신, 지역 별 상황 조회 | 웹 또는 모바일 |

### 3.2 기술 스택 (간결화)

| 계층 | 기술 | 용도 |
|------|------|------|
| **모바일 앱** | Kotlin + Android Studio + Pleos Connect SDK | 위험신호 발송, 순찰차 알림 |
| **Backend** | Node.js/Express 또는 Python/FastAPI | REST API, 데이터 처리 |
| **데이터베이스** | PostgreSQL | 신고, 위치, 상태 데이터 |
| **실시간 통신** | Firebase FCM + WebSocket | 푸시 알림, 실시간 위치 동기화 |
| **웹 대시보드** | React.js + Google Maps API | 통합 모니터링 화면 |
| **클라우드** | AWS (EC2, RDS) 또는 GCP | 서버 배포 및 호스팅 |
| **보안** | JWT, SSL/TLS, 역할 기반 접근 제어 (RBAC) | 인증, 암호화, 권한 관리 |

### 3.3 시스템 데이터 흐름

```
[사용자 모바일 앱]
    ↓ (위험신호 + GPS + 사진)
[Backend API Server]
    ↓ (검증 및 처리)
[PostgreSQL Database]
    ↓ (신고 기록 저장)
[FCM Push Notification]
    ↓ (즉시 알림)
[순찰 차량 앱] → [웹 대시보드]
    ↓ (상태 업데이트)
[Backend] → [Dashboard Real-time Update]
```

### 3.4 Pleos SDK 핵심 모듈 활용

| 모듈 | 활용 기능 |
|------|---------|
| **Fused Location SDK** | 순찰 차량의 실시간 GPS 위치 추적 |
| **Sync SDK** | 신고 데이터 및 위치 정보 실시간 동기화 |
| **Vehicle SDK (선택)** | 차량 상태 정보 (외부 온도 등) 수집 (Phase 2 확장) |

## 7. 부록: 상세 기술 사양 및 확장 계획

### 7.1 상세 기술 사양 (참고용)

#### 7.1.1 Android 앱 아키텍처

```kotlin
// 위험신호 전송 기본 구조
class EmergencyReportActivity {
    fun sendEmergencySignal() {
        val location = getCurrentLocation() // Fused Location SDK
        val report = EmergencyReport(
            title = editText.text.toString(),
            description = editText2.text.toString(),
            latitude = location.latitude,
            longitude = location.longitude,
            imageUri = capturedImageUri,
            timestamp = System.currentTimeMillis()
        )
        
        // Backend에 전송
        apiClient.post("/api/emergency", report)
            .onSuccess { 
                showToast("신고가 접수되었습니다")
                startLocationTracking() // 순찰차에서 위치 추적
            }
    }
}
```

#### 7.1.2 Backend API 엔드포인트

```
POST   /api/emergency                 - 위험신호 등록
GET    /api/emergency/{id}            - 신고 상세 조회
PUT    /api/emergency/{id}/status     - 신고 상태 업데이트
GET    /api/patrol/location/{id}      - 순찰차 위치 조회
POST   /api/notification/send         - FCM 푸시 알림 발송
```

#### 7.1.3 데이터베이스 스키마 (주요 테이블)

```sql
-- 긴급 신고
CREATE TABLE emergency_reports (
    id UUID PRIMARY KEY,
    user_id VARCHAR NOT NULL,
    title VARCHAR NOT NULL,
    description TEXT,
    latitude DOUBLE PRECISION,
    longitude DOUBLE PRECISION,
    image_url VARCHAR,
    status VARCHAR (NEW, ASSIGNED, IN_PROGRESS, COMPLETED),
    created_at TIMESTAMP,
    updated_at TIMESTAMP
);

-- 순찰 차량 위치 기록
CREATE TABLE patrol_locations (
    id UUID PRIMARY KEY,
    vehicle_id VARCHAR NOT NULL,
    latitude DOUBLE PRECISION,
    longitude DOUBLE PRECISION,
    timestamp TIMESTAMP,
    INDEX (vehicle_id, timestamp)
);

-- 신고-순찰차 할당
CREATE TABLE emergency_assignments (
    id UUID PRIMARY KEY,
    emergency_id UUID,
    vehicle_id VARCHAR,
    assigned_at TIMESTAMP,
    arrival_time TIMESTAMP
);
```

### 7.2 확장 계획 (향후 추진)

#### 7.2.3 장기 확장 (12개월 이상)
- **다중 캠퍼스 확대**: 타 대학, 산업단지, 스마트 시티 확산

- **국제 표준화**: ISO 27001(보안), ISO 9001(품질) 취득

### 7.3 데이터 보안 및 개인정보 보호

**보안 정책**
- 모든 API 통신은 HTTPS/TLS 1.3 이상 암호화
- JWT 토큰 기반 사용자 인증 (만료 시간 1시간)
- 역할 기반 접근 제어 (RBAC): 사용자/순찰차/관리자 별 권한 분리

**개인정보 처리**
- 신고자 개인정보는 암호화 저장 및 최소 1년 후 삭제
- 위치 데이터는 현장 대응 완료 후 30일 후 삭제
- GDPR/개인정보보호법 준수

- **즉각적 안전 대응**: 위험신호 → 순찰차 배치 → 현장 도착 평균 5분 이내
- **기술 혁신**: 자동차 OS 기술의 캠퍼스 안전 도메인 응용
- **인력 양성**: 모빌리티 산업 실무 경험 축적
- **산업 협력**: 현대자동차와의 기술 협력 및 공모전 수상 기회
- **사회 기여**: 캠퍼스 안전 문화 확산

이 프로젝트의 성공적 추진은 팀의 기술 역량 강화, 학교의 혁신 이미지 제고, 그리고 현실적 안전 관리 체계 구축이라는 다층적 가치를 동시에 실현할 수 있습니다.
